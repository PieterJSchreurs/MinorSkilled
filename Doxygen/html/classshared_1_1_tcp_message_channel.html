Detailed Description

TcpMessageChannel is sort of a facade around a TcpClient, Packets & the StreamUtil class. It abstracts communication to a single bidirectional channel which you can use to pass ASerializable objects back and forth, and check to see whether everything is still peachy where the underlying connection is concerned.

Basically after the initial setup, 'all' you have to worry about is having a channel and being able to push objects through it.

If you want to implement your own serialization mechanism, this is the place to do it.
